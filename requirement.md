메인 스크린에는 1. 와이드 캐러셀 컴포넌트, 2. 카드 캐러셀 컴포넌트, 3. 콜아웃 컴포넌트를 구현할 것이다.

페이지랑 라우터랑 폴더구조생각 x , 컴포넌트들만 먼저 다 만들어서 각각 1번페이지 2번페이지 생성해서 그냥 컴포넌트 테스트만하셈 딴거하지말고

만든 컴포넌트가 화면이랑 똑같이 나오는지, 반응형쪽만 테스트하고

모바일 반응형이면 거기서 이제 다 비슷하게 나오는지만 테스트하기

image.png, image copy.png, image copy 2,3,4,5,6.png 와 같이 있는 스크린들에 공통으로 들어갈 그리드 컴포넌트를 구현해야한다. 컨텐츠 내용이 늘어날수록 3x(width)에서 계속 늘어나는 그리드형태가 되어야하며, 다른 것에 집중하지 지않고 그리드 컴포넌트를 완벽히 이 스크린들에 컴포넌트 재활용 목적만을 충실하여 작업되어야한다.

inputScreen.png , inputScreen2.png 두개의 스크린에 있는 input 컴포넌트를 재활용하는 목적에 충실하여 제작한다

modalScreen, modalScreen2, modalScreen3.png 세개의 스크린에 있는 모달 컴포넌트의 목적으로 쓰일 컴포넌트를 재활용하는 목적에 충실하여 제작한다.


모든 스크린의 스타일은 보여준 이미지 파일들에 완벽히 일치하여야한다.
