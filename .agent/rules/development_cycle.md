# 개발 프로세스 및 에이전트 행동 수칙

이 문서는 프로젝트 개발 시 모든 에이전트가 엄격히 따라야 할 개발 사이클과 규칙을 정의합니다. 에이전트는 이 기준을 벗어나는 행위를 해서는 안 됩니다.

## 1. 기획안 기반 PRD 생성 (PRD Generation)
- **Action**: 제공된 기획안을 바탕으로 구체적인 **PRD(제품 요구사항 정의서)**를 생성한다.
- **Goal**: 개발 시작 전 명확한 목표와 범위를 확정한다.

## 2. 요구사항의 세분화 (Decomposition)
- **Action**: PRD의 요구사항들을 **작은 기능(Small Feature)** 단위로 쪼갠다.
- **Definition**: '작은 기능'이란 요구사항을 충족하기 위해 필요한 **하나의 행위(Activity)**를 의미한다.

## 3. 데모 프로젝트 생성 (Demo Project Creation)
- **Rule**: 메인 프로젝트에 바로 코드를 작성하지 않고, 각 작은 기능별로 **별도의 데모 프로젝트**를 생성하여 구현 및 검증한다.
- **Types of Demos**:
  - **Feature**: 핵심 로직, 함수(Function), 클래스(Class) 단위의 기능 구현.
  - **Style**: UI 컴포넌트, 레이아웃 등 시각적 요소 구현.
  - **Interaction Design**: 사용자의 상호작용에 따른 기능(Function)과 스타일(Stylesheet) 구현.

## 4. 메인 프로젝트 통합 (Integration)
- **Action**: 완성되고 검증된 데모 프로젝트들을 하나씩 참조(Reference)하며 **메인 프로젝트**에 기능을 통합한다.
- **Method**: 데모에서 검증된 로직과 스타일을 가져와 체계적으로 합친다.
